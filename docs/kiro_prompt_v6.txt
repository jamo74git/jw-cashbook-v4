CORE RULES:

NON-NEGOTIABLE MODULES:

1. AUTH + DASHBOARD ROUTING
   Login checks user_hierarchy_access. Route to role dashboard per scope.

2. TREASURER CAPTURE FLOW - SERVICE LEVEL
   Step 1: Select Congregation + ServiceDate + ServiceType[AM,PM]
   Step 2: Capture Member Tithing per Priestship.officer_code. Fields: Amount, IncomeType[Cash,EFT,DD]
           Rule: If IncomeType != Cash then photo_upload REQUIRED.
   Step 3: Capture Officer Tithing per individual officer_code.
   Step 4: Balance check: income must = banking + expenses. Then SUBMIT SERVICE FOR AUDIT. Status=PENDING_AUDIT

3. AUDIT WORKFLOW - CONGREGATIONAL LEVEL ONLY
   a. 2x AUDITORS assigned per Congregation. Review PENDING_AUDIT. Actions: AUDIT_APPROVE or AUDIT_REJECT + comment.
   b. CHAIRPERSON can ASSUME Treasurer or Auditor role with O override. All assumption actions audited as SELF_REVIEW_EXCEPTION.
   c. When ALL services in a month are AUDIT_APPROVED, ELDER can SUBMIT MONTH TO OVERSEER.
   d. OVERSEER reviews and OverseerReviewApprove or OverseerReviewReject + comment.
   e. If OVERSEER_APPROVE -> OVERSEER SUBMITTED_TO_HO. 
   f. If REJECTED at any stage -> Status=DRAFT. Congregational Management Committee[Treasurer, 2xAuditors, Chair] resolves.

4. CENSUS MODULE - PRIESTSHIP HEALTH
   Captured by: TREASURER or ELDER per Congregation
   Per Priest: eligible_to_tithe, children, youth, adults, seniors, total_members, last_updated_date
   FLAGS: 90 days no update = ORANGE. 180 days = RED on Priest dashboard.
   VISIBILITY: Chair/Elder=Cong Total. Overseer/Apostle=Overseership Total. HO=Conference Total.
   SECRETARY = NO ACCESS TO CENSUS.

5. BULK ADMIN FOR HO

   Respect ho_district_assignments. Validate + Audit log all bulk actions.
